Architecture

How It Works

Everything runs locally in your browser. No data leaves your machine.

01

Page Load

When you navigate to any http/https page, Chrome injects the content scripts automatically at document_idle.

02

Run Checks

11 modules run 55 check functions against the live DOM. Each returns a result with severity, label, and detail.

03

Store + Badge

Results are stored in chrome.storage.local. The badge on the extension icon shows the issue count.

04

Popup

Click the icon. Filter by severity (Critical / Warning / Passed), browse by category tab. Every issue shows actionable detail.

Severity Levels

How Issues Are Classified

Every check returns one of three severity levels. The badge color reflects the worst issue found.

Critical

Blocks indexing, breaks rendering, or causes major SEO damage. Must fix. Badge turns red.

Warning

Hurts rankings, accessibility, or performance but page still functions. Should fix. Badge turns orange.

Passed

Check ran and found no issues. Confirms best practices are met. Shown under the green "Passed" filter.

Security

Minimal Permissions

The extension requests only what it needs. No data leaves your browser except optional GSC API calls.

Required

storage: save check results locally
tabs: read active tab URL for badge
identity: Google OAuth for GSC

Optional (user-granted)

host_permissions: only for HTTP link probe
User must click "Allow" explicitly
Never auto-requested

Never Used

No activeTab, no scripting
No webRequest, no remote scripts
No eval(), no innerHTML with data

Security

Built Secure by Default

Every line of code follows strict security rules. These are not optional guidelines, they are enforced throughout the codebase.

XSS Prevention

Zero innerHTML with Dynamic Data

All UI rendering uses .textContent and createElement. No page data ever touches innerHTML.

No Dynamic Code

No eval(), No new Function()

The extension never executes strings as code. CSP is never relaxed. No unsafe-inline or unsafe-eval.

Input Validation

Nothing from the Page Is Trusted

All strings are type-checked, capped in length (URLs: 500, labels: 80, details: 400), and sanitized before use.

Storage Safety

Capped, Typed, Static Keys

Max 100 results. Hardcoded storage keys. No raw page content, no cookies, no PII ever stored.

Message Security

Validated Both Directions

Background verifies sender.id on every message. Types whitelisted with a Set. Unknown types silently rejected.

No Remote Code

100% Local JavaScript

All scripts bundled locally. No CDNs, no remote fetches for code. Only outbound calls: optional link probes and GSC API.

Least Privilege

Minimum Permissions

Only 3 permissions. Host access for link probes is optional and user-granted. No broad permissions ever.

Data Privacy

Nothing Leaves Your Browser

All checks run locally. No analytics, no telemetry, no tracking. GSC is opt-in and calls Google's API directly.

Strict Mode Everywhere

Every JavaScript file starts with 'use strict'. No implicit globals, no silent type coercion, no scope leaks. Combined with const by default and === everywhere.
